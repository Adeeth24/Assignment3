      • git status Show the working tree status.
      It examines your working directory and Git repository
        It tells you which files are:
      • Untracked: New files that haven't been added to Git.
      • Modified: Files that have been changed since the last commit.
      • Staged: Files that have been added to the staging area (ready for the next commit).


    • git add:- The files are added in a temporary holding area called as staging area. This is used to add specific file

      git add . :-This is used when we have to add all files in one go to staging area.


    • git commit:- Takes the changes made in staged using git add and then creates a permanent record of them in your Git repository history

      git commit -m '_______name':- (-m)Flag combination is a convinent way to commit your changes in Git along with a commit message, all in one command.

      It initiates the commit process, capturing a snapshot of your current project state.

      1. git remote add origin https://github.com/Adeeth24/Assignment3.git It adds a remotr repository URL to your local Git repository.
        remote: This tells git you're working with a remote repository.
        add: This specifies that you're adding a new remote.
        origin: This is a common name given tothe main remain repository you're working with in a project. you can choose any name

      2. git branch -M mainThis command renames your current local branch to main.
        git branch: This command is used to manage local branches.
        -M: This flag tells Git to move (rename) the current branch.
        main: This is the new name you want to assign to your current local branch.

      3. git push -u origin main This command pushes your local main branch to the remote repository you configured in step 1.
        -u: This flag sets the upstream branch for your local branch. This means future pushes to your local main branch can be done with just git push
        git push without specifying the remote and branch name again.
        origin: This refers to the remote repository alias you configured in step 1 (or the name you chose).
        main: This specifies the local branch you want to push to the remote repository.
